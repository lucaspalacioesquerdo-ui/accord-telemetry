DEPLOY GUIDE

Accord Telemetry · Next.js + Supabase + Vercel · ~15 minutos
PASSO 1
Criar conta no Supabase ~3 min
O banco de dados que guarda seus logs
  1. Acesse supabase.com e clique em Start your project
  2. Faça login com sua conta GitHub (mais fácil)
  3. Clique em New project
  4. Escolha um nome (ex: accord-telemetry) e região: South America (São Paulo)
  5. Defina uma senha para o banco (guarda em lugar seguro) → clique Create new project
  6. Aguarde ~2 minutos para o projeto ser criado
PASSO 2
Criar a tabela no Supabase ~2 min
Roda o SQL que cria a estrutura do banco
  1. No painel do Supabase, clique em SQL Editor no menu lateral (ícone de código)
  2. Clique em + New query
  3. Abra o arquivo supabase_schema.sql desta pasta com qualquer editor de texto
  4. Copie todo o conteúdo e cole no SQL Editor
  5. Clique em Run (ou Ctrl+Enter)
  6. Deve aparecer "Success. No rows returned" — está certo
A tabela log_sessions foi criada. Ela vai armazenar as métricas de cada CSV que você importar.
PASSO 3
Copiar as credenciais do Supabase ~1 min
Você vai precisar dessas duas chaves no próximo passo
  1. No painel do Supabase, clique em Settings (ícone de engrenagem) → API
  2. Copie a Project URL — parece com: https://abcdefgh.supabase.co
  3. Copie a anon / public key — começa com eyJ...
Não compartilhe essas chaves publicamente. A anon key é segura para uso no front-end, mas não a service_role.
PASSO 4
Subir o código no GitHub ~3 min
O Vercel vai ler direto do seu repositório
  1. Acesse github.com/new e crie um repositório chamado accord-telemetry
  2. Pode ser privado — o Vercel acessa mesmo assim
  3. Faça upload desta pasta completa. Você tem duas opções:

Opção A — pelo site do GitHub (sem terminal):

  1. No repositório criado, clique em uploading an existing file
  2. Arraste todos os arquivos e pastas desta pasta para lá
  3. Clique Commit changes

Opção B — pelo terminal:

# Dentro da pasta accord-telemetry git init git add . git commit -m "initial commit" git remote add origin https://github.com/SEU_USER/accord-telemetry.git git push -u origin main
PASSO 5
Deploy no Vercel ~3 min
O app vai ficar online com URL pública
  1. Acesse vercel.com e faça login com GitHub
  2. Clique em Add New → Project
  3. Selecione o repositório accord-telemetry e clique Import
  4. Em Environment Variables, adicione as duas variáveis:
NEXT_PUBLIC_SUPABASE_URL = https://SEU_ID.supabase.co NEXT_PUBLIC_SUPABASE_ANON_KEY = eyJ...sua_chave_aqui
  1. Clique Deploy
  2. Aguarde ~2 minutos
  3. Pronto! Você receberá uma URL tipo accord-telemetry.vercel.app
A cada novo push no GitHub, o Vercel faz deploy automático. Você nunca precisa mexer no Vercel de novo.
COMO USAR
Importando CSVs do HondsH
Fluxo de trabalho depois do deploy
  1. Abra sua URL do Vercel no celular ou computador
  2. Arraste um ou mais arquivos .csv do HondsH na sidebar
  3. O app extrai as métricas automaticamente e salva no banco
  4. A linha do tempo atualiza incluindo o novo log
  5. Na próxima vez que abrir, todos os logs anteriores já estão lá
O app funciona com CSVs em inglês e português — o HondsH exporta nas duas línguas dependendo da versão, e o parser detecta automaticamente.
ATUALIZAR O APP
Quando eu atualizar o código para você
Processo para receber melhorias futuras

Quando eu gerar uma versão nova do projeto, você:

  1. Substitui os arquivos no seu repositório GitHub
  2. O Vercel faz o re-deploy automaticamente em ~2 minutos
  3. Os dados no Supabase são preservados — nenhum log é perdido
Os dados ficam separados do código. Você pode atualizar o app sem perder nenhum histórico.
